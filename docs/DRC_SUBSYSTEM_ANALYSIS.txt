1. OVERALL ARCHITECTURE
================================================================================

The DRC (Design Rule Check) engine is production-grade and comprehensive, 
supporting the SKY130 130nm open-source PDK with 224 built-in rules. It features:

- Spatial indexing for O(n log n) rule checking performance
- JSON-based external PDK rule deck loading/export
- Flexible rule types with conditional/width-dependent parameters
- Industrial severity levels (ERROR, WARNING, INFO)
- Support for 5-metal layer stack (MET1-MET5) plus front-end layers

  • width >= 0.42 um AND PRL > 0.45 um  →  spacing = 0.220 um

MET3 PRL Spacing:
  • width >= 0.90 um AND PRL > 0.90 um  →  spacing = 0.450 um

MET4 PRL Spacing:
  • width >= 0.90 um AND PRL > 0.90 um  →  spacing = 0.450 um

Implementation:
  - parallel_run_length(WireSegment a, WireSegment b) calculates PRL
  - Uses condition_min_width for width threshold
  - Uses condition_min_length for PRL threshold
  - check_parallel_run() evaluates both conditions

Algorithm: Lines 36-49 in drc.cpp - computes overlap in parallel direction
by finding min/max coordinates along common axis, returns max(0, overlap).

14. DATA STRUCTURES & KEY ALGORITHMS
================================================================================

Core Data Structure (DrcRule):
  name: rule identifier
  description: human-readable description
  value: primary parameter (um or ratio)
  type: rule category enum
  layer, layer2: layer assignments
  aux_value, aux_value2: secondary parameters
  condition_min_width: width threshold for conditional rules
  condition_min_length: PRL/length threshold for conditional rules
  severity: ERROR/WARNING/INFO
  enabled: on/off flag

Spatial Index:
  - Grid cells: sgrid_nx_ × sgrid_ny_ cells
  - Per-cell: wire_ids, cell_ids, via_ids vectors
  - Query radius: scaled by rule value (e.g., rule.value * 5 + wire_length)

Key Algorithms:

parallel_run_length(WireSegment a, WireSegment b):
  1. Check orientation: both horizontal or both vertical?
  2. If different → return 0
  3. If horizontal: overlap on x-axis = min(a1,b1) - max(a0,b0)
  4. If vertical: overlap on y-axis = min(a1,b1) - max(a0,b0)
  5. Return max(0, overlap) [can't be negative]

wires_spacing(WireSegment a, WireSegment b):
  1. Convert wires to rectangles (center ± width/2)
  2. Compute gap in x: max(0, max(Ra.x0-Rb.x1, Rb.x0-Ra.x1))
  3. Compute gap in y: max(0, max(Ra.y0-Rb.y1, Rb.y0-Ra.y1))
  4. Return Euclidean distance: sqrt(dx² + dy²)

check_conditional_spacing():
  For each CONDITIONAL_SPACING rule:
    For each wire on rule.layer:
      If wire.width >= rule.condition_min_width:
        For nearby wires:
          Calculate PRL (parallel_run_length)
          If PRL >= rule.condition_min_length:
            Check spacing < rule.value → violation
